🎯 **What:** The `CategoryDetailGenerator` contained an untested `except` block during initialization where it gracefully fell back to `None` if `GeminiRestClient` failed to initialize (e.g., due to missing API keys or network errors). This PR adds a unit test targeting this specific exception handling.
📊 **Coverage:** A new test `test_init_exception_handling` correctly mocks `GeminiRestClient`, raises a mock exception during instantiation, and asserts that the `gemini_client` is safely set to `None`. Additionally, `pydantic` was added to the `sys.modules` mock list to ensure the test suite is capable of running smoothly without throwing `ModuleNotFoundError` inside constrained environments where standard dependencies might be missing.
✨ **Result:** Increased test coverage by thoroughly validating the failure pathway inside the initialization constructor.
